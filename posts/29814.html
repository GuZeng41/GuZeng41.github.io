USACO Training 1.5.4 Checker Challenge

Description

检查一个如下的6 x 6的跳棋棋盘，有六个棋子被放置在棋盘上，使得每行、每列只有一个，每条对角线(包括两条主对角线的所有对角线)上至多有一个棋子。

上面的布局可以用序列2 4 6 1 3 5来描述，第i个数字表示在第i行的相应位置有一个棋子，如下：

行号 1 2 3 4 5 6

列号 2 4 6 1 3 5

这只是跳棋放置的一个解。请编一个程序找出所有跳棋放置的解。并把它们以上面的序列方法输出。解按字典顺序排列。请输出前3个解。最后一行是解的总个数。

Input

一个数字N (6 <= N <= 13) 表示棋盘是N x N大小的。

Output

前三行为前三个解，每个解的两个数字之间用一个空格隔开。第四行只有一个数字，表示解的总数。

Sample Output 1

Hint

特别注意: 对于更大的N(棋盘大小N x N)你的程序应当改进得更有效。 不要使用打表法

题解：本题是一道经典的dfs好题。思路为：从第一行开始放，放一个位置，判断合法之后放下一行，直到放完n行为止。

思路是很简单，但是我踩了挺多坑的。

剪枝采用的思路是直接循环判断坐标是否在同一行列对角线上，复杂度较高

事实上刚刚写的时候还踩了坑

首先是我在代码中注释位置写上给l数组（记录某一行放的是第几列，也就是x坐标）在每一种正确情况后面都将他们归零，显然这是不正确的，为什么呢？因为l数组中还存储着前面的存放位置坐标，在下一次从头开始放的过程中，我们需要上次的数据来判断总次数是不是需要加一。极端点说，如果你把整个数组归零，那么第二次做的时候做出和第一次一样的答案，cnt不是也会++嘛？

第二个坑的话就是check里面循环的限制了

for(int i=1;i<y;i++)这是修正之后的代码，代表检索他前面行有没有重复的，而我一开始愚蠢的写了n。。与后面的有什么相干？？

到这里你以为完了嘛？这个代码并不能AC，最后一个测试点TLE了。。所以我不得不对其进行优化。

然而怎么说，还有一个需要注意的地方，那就是

for(int i=1;i<=n;i++)cout<<l[i]<<" "; cout<<endl;

这样子，会在行末多输出一个空格，而这样很可能会WA掉，下次务必注意输出格式

然而还是T了

这里尝试第一次改进：运用桶的思想，把出现过的行列对角线标记为True。

主对角线：eg：(1,1),(2,2),(3,3)是差相等

副对角线则是和相等，但要注意下标不能负数，所以可以整体加个n进行坐标偏移。

这里有个小细节要说一下，回到上一步清除标记为什么要写在dfs(r+1)下面呢？首先我在return 上的理解有误差，一开始把flag归0写在了cnt++那边，但这显然是不正确的，其实在进入dfs(r+1)之后，他有两种情况返回dfs(r+1)的位置：层层推进，当走到最后一行，完成一种情况时，由我写的return 退回；或者是在推进过程中的某一次循环全部做完了，在dfs函数底部自带了一个return回到上一层递归

USACO Training 1.5.1 Number Triangles

Description

观察下面的数字金字塔。

写一个程序来查找从最高点到底部任意处结束的路径，使路径经过数字的和最大。每一步可以走到左下方的点也可以到达右下方的点。

在上面的样例中,从7 到 3 到 8 到 7 到 5 的路径产生了最大和:30

Input

第一个行包含 R(1<= R<=1000) ,表示行的数目。

后面每行为这个数字金字塔特定行包含的整数。

所有的被供应的整数是非负的且不大于100。

Output

单独的一行包含那个可能得到的最大的和。

Sample Input 1

Sample Output 1

这题是一题经典的dp入门题，我可以说是没有接触过dp，想了很久菜做出来，实在是菜的抠脚啊。。。

首先一个灵魂拷问：这题能用贪心做吗？即从第一个位置往下走，找出下方两个中较大的相加，并一路走到底呢？？

这是个很容易跳的坑，答案是不能！原因很简单，最大和不一定是每一个数都最大，举个例子，最左边一列为1，1，1，1，99999999，按贪心算法是做不到这一支的，自然不可能对。

那么应该怎么做呢？

分析：题目要求输出最大的和，而我们应该要求出每一行中和的最大数，一直累加到最后一行。而每个数都可以加到下一行的左右下方两数，那么反过来说，得到每一行的最大和可以从第二行做起，每个数都是由上两个对应数相加，在每一行中取 max，最后输出答案。

USACO Training 1.5.3 Superprime Rib

Description

农民约翰的母牛总是生产出最好的肋骨。你能通过农民约翰和美国农业部标记在每根肋骨上的数字认出它们。

农民约翰确定他卖给买方的是真正的质数肋骨,是因为从右边开始切下肋骨,每次还剩下的肋骨上的数字都组成一个质数,举例来说:

7 3 3 1

全部肋骨上的数字 7331是质数;三根肋骨 733是质数;二根肋骨 73 是质数;当然,最后一根肋骨 7 也是质数。

7331 被叫做长度 4 的特殊质数。

写一个程序对给定的肋骨的数目 N (1<=N<=8),求出所有的特殊质数。数字1不被看作一个质数。

Input

单独的一行包含N。

Output

按顺序输出长度为 N 的特殊质数,每行一个。

Sample Input 1

Sample Output 1

绝了，我第一想法就是打表，我没救了

打表的时候遇到个问题：我把答案记录在vector套vector中，那么如何输出这些数呢？

for(auto x:v[n]) cout<<x;

for(int i = 0; i <v[1].size(); I++) cout<<v[1][i];

当然平时训练的时候最好还是不要打表了，这题还有什么做法呢？很容易就能想到深搜。为什么呢？一个数字，一位一位递进（和递退是等价的）dfs的代码一般都很简洁：思路为从第一位开始，往后填数字，但是由于后面的质数末尾肯定为奇数，所以只需循环计数的情况就可以了。

搜索还得好好练那。

但其实这题还有解法。

这题适合用STL queue做，思路和上解相同，先生成母数，将母数放在队首，将拓展位数后合法的数push到队尾，结束后再pop队首的母树。
