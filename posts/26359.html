P1212 [USACO1.4]铺放矩形块 Packing Rectangles

题目描述

给定 44 个矩形块，找出一个最小的封闭矩形将这 44 个矩形块放入，但不得相互重叠。所谓最小矩形指该矩形面积最小。

44 个矩形块中任一个矩形的边都与封闭矩形的边相平行，上图显示出了铺放 44 个矩形块的 66 种方案。

这 66 种方案是唯一可能的基本铺放方案。因为其它方案能由基本方案通过旋转和镜像反射得到。

可能存在满足条件且有着同样面积的各种不同的封闭矩形，你应该输出所有这些封闭矩形的边长。

输入格式

共有 44 行，每行两个正整数，表示每个矩形的边长。

输出格式

总行数为解的总数加一。

第一行是一个整数，代表封闭矩形的最小面积。
接下来的每一行都表示一个解，由 p,q\space (p \leqslant q)p,q (p⩽q) 来表示。这些行必须根据 pp 的大小按升序排列，且所有行都应是不同的。

输入输出样例

输入 #1复制

输出 #1复制

说明/提示

【数据范围】
对于 100%100% 的数据，输入的所有数在 [1,50][1,50] 内。

题目翻译来自NOCOW。

USACO Training Section 1.4

题解：从现在开始，题目变得有些难了。本周都是模拟题。虽说思路方面会比一些算法题目来得简单，但是在暴力模拟的过程中细节的处理变得很重要，尤其是我这种刚刚接触ACM的人来说，在做模拟题的过程中积累一些常用的语言小技巧，也是很有必要的。

说了这么多废话，先开始上思路。题目中木板个数是固定的4块，摆放方式一共六种（实际上四五是同一种）

他要求输出全部的可能矩形面积以及具体尺寸，那么就要求每一个答案都必须记录和比较。

下面正式进入分析：

输入

每块木板有长和宽两个属性，需要一个存储两个属性的“容器”。这里我觉的最合适的是结构体和pair。

这里选择了结构体，原因是我对它还不够熟悉，想拿来练练手。

处理

获取了木板 的两个属性之后，将其和前面六种情况进行一一比对模拟，最终记录最小面积以及具体尺寸。这里注意，每块木板的标号可以改变，所以需要定义一个数组专门存储下标，并进行全排列遍历。（其实这里应该可以优化，比如对木板的长宽进行排序，减少讨论，但这样的尝试暂时没有成功。。）思路看起来很简单，但是这题对我来说真的很困难。。

USACO Training 1.4.3 Arithmetic Progressions

Description

一个等差数列是一个能表示成a, a+b, a+2b,…, a+nb (n=0,1,2,3,…)的数列。

在这个问题中a是一个非负的整数，b是正整数。写一个程序来找出在双平方数集合S中长度为n的等差数列。双平方数集合是所有能表示成p^2＋q^2的数的集合。

Input

第一行: N(3<= N<=25),要找的等差数列的长度。

第二行: M(1<= M<=250),搜索双平方数的上界0 <= p,q <= M。

Output

如果没有找到数列,输出`NONE’。

如果找到了，输出一行或多行, 每行由二个整数组成:a,b。

这些行应该先按b排序再按a排序。

所求的等差数列将不会多于10,000个。

Sample Input 1

Sample Output 1

思路：这题我觉得很棒，首先有两个大思路：到底是在双平方数的集合中寻找等差数列呢，还是在等差数列中搜索双平方数。最后的答案是后者。

原因是什么呢？让我们来思考一下，如果在双平方数集合中寻找等差数列，因为等差数列并不一定是连续的，也就是步长不知道，你很难通过对称位置的和相等来寻找等差数列随后就只能在等差数列中寻找双平方数。实现代码时，我本来挺自信的选择了set作为存储双平方数集合的容器，但现实很残酷——T了。set的确具有去重/排序等优点，但他每一次的查找，排序都需要nlogn的时间，实在是顶不住啊！所以这题适合另一种思路：桶的思想。因为MAX=
$$
2×250×250
$$
可以开如上大小的桶，标记每一个数字是否出现过。具体代码的话就是三重循环，进行适当的剪枝

P1215 [USACO1.4]母亲的牛奶 Mother’s Milk

题目描述

农民约翰有三个容量分别是 a,b,ca,b,c 升的桶。

最初，a,ba,b 桶都是空的，而 cc 桶是装满牛奶的。有时，农民把牛奶从一个桶倒到另一个桶中，直到被灌桶装满或原桶空了。

当然每一次灌注都是完全的。由于节约，牛奶不会有丢失。

写一个程序去帮助农民找出当 aa 桶是空的时候，cc 桶中牛奶所剩量的所有可能性。

输入格式

单独的一行包括三个整数 a,b,c。

输出格式

只有一行，升序地列出当 a 桶是空的时候，c 桶牛奶所剩量的所有可能性。

输入输出样例

输入 #1复制

输出 #1复制

输入 #2复制

输出 #2复制

【数据范围】
对于 100% 的数据1≤a,b,c≤20。
